# Plan 093: NLL v2 — Language & Ergonomics

**Priority:** High
**Effort:** 5-7 days
**Depends on:** None
**Target:** `crates/nlink-lab/src/parser/nll/`, `bins/lab/`

## Summary

A single cohesive upgrade to the NLL language covering expressions, syntax
sugar, and tooling. All changes target the same files (lexer, parser, AST,
lower) and share the same test infrastructure, so they're most efficiently
implemented as one unit.

Grouped into three waves by dependency order:

- **Wave 1** (days 1-2): Foundation — expression engine, block comments, new tokens
- **Wave 2** (days 3-4): Syntax sugar — subnet auto-assign, list iteration, defaults, for-expressions
- **Wave 3** (days 5-6): Tooling — lab metadata, render command
- **Buffer** (day 7): Integration testing, documentation, edge cases

## Breaking Changes

**ForLoop AST**: `start: i64, end: i64` fields replaced by `range: ForRange`
enum. This is an internal-only change — no NLL syntax breaks.

All NLL syntax changes are backward-compatible additions.

## Wave 1: Expression Engine (days 1-2)

Everything in this wave touches `lower.rs` (expression evaluation) and
`lexer.rs` (block comments). No AST or parser changes needed.

### 1.1 Modulo Operator

**Current**: `eval_expr()` at `lower.rs:283-339` supports `+`, `-`, `*`, `/`.

**Change**: Add `%` to the operator list at line 287 and match arm at line 317.

```rust
// lower.rs:287 — add '%' to scan list
for &op in &['+', '-', '*', '/', '%'] {
// lower.rs:317 — add match arm
'%' => { if right_val == 0 { return format!("${{{expr}}}"); } left_val % right_val }
```

**Tests**: `${i % 2}` → `0` or `1`, `${i % 3}`, `${4 % 0}` → unchanged.

### 1.2 Compound Expressions

**Problem**: Only single binary operations work. `${(i - 1) * 2}` fails.

**Change**: Replace `eval_expr()` with a recursive-descent mini parser:

```
expr   = term (('+' | '-') term)*
term   = factor (('*' | '/' | '%') factor)*
factor = '(' expr ')' | variable | integer
```

```rust
fn eval_expr(expr: &str, vars: &HashMap<String, String>) -> String {
    let toks = tokenize_expr(expr.trim());
    match parse_arith(&toks, &mut 0, vars) {
        Ok(val) => val.to_string(),
        Err(_) => format!("${{{expr}}}"),
    }
}

// tokenize_expr: split into numbers, identifiers, operators, parens
// parse_arith: precedence-climbing for +- vs */%
// parse_factor: parens, var lookup, literal int
```

~60 lines replacing the current ~55 lines. All existing expressions still work.

**Tests**: `${(i - 1) * 2}`, `${a + b + c}`, `${i % 2 + 1}`, `${(i + 3) / 2}`.

### 1.3 Conditional Expressions

**Change**: Add ternary operator to `eval_expr()`. Check for `?` before
arithmetic parsing:

```rust
// If expression contains '?', treat as ternary
if let Some(q) = find_ternary_question(expr) {
    let cond = &expr[..q].trim();
    let rest = &expr[q + 1..];
    let colon = rest.find(':').ok_or("missing :")?;
    let true_val = resolve(rest[..colon].trim(), vars);
    let false_val = resolve(rest[colon + 1..].trim(), vars);
    let result = eval_condition(cond, vars); // supports ==, !=
    return if result { true_val } else { false_val };
}
```

Conditions: `var == "literal"`, `var != "literal"`, `var == var2`.

**Tests**: `${env == "prod" ? "5ms" : "50ms"}`, `${i != 0 ? "yes" : "no"}`.

### 1.4 Contextual Auto-Variables

**Change**: Inject variables in `expand_for()` (lower.rs:216-246) and at the
lowering entry point.

Loop variables (injected per iteration):

| Variable | Value |
|----------|-------|
| `${loop.index}` | Current iteration value (same as loop var) |
| `${loop.first}` | `"true"` on first iteration |
| `${loop.last}` | `"true"` on last iteration |

Lab variables (injected once before statement processing):

| Variable | Value |
|----------|-------|
| `${lab.name}` | Lab name |
| `${lab.prefix}` | Lab prefix or name |

Clean up `loop.*` vars after loop completes.

### 1.5 Block Comments

**Change**: Pre-process input to strip `/* ... */` before lexing. Support nesting.
Preserve line numbers by replacing comment content with newlines.

```rust
fn strip_block_comments(input: &str) -> Result<String> {
    let mut result = String::with_capacity(input.len());
    let mut depth = 0;
    let mut chars = input.chars().peekable();
    while let Some(c) = chars.next() {
        if c == '/' && chars.peek() == Some(&'*') { chars.next(); depth += 1; }
        else if c == '*' && chars.peek() == Some(&'/') && depth > 0 { chars.next(); depth -= 1; }
        else if depth == 0 { result.push(c); }
        else if c == '\n' { result.push('\n'); }
    }
    if depth > 0 { return Err(Error::NllParse("unterminated block comment".into())); }
    Ok(result)
}
```

Call in `lex()` before `Token::lexer()`.

## Wave 2: Syntax Sugar (days 3-4)

These changes touch the AST, parser, and lowering together. They share the
pattern of "parse new syntax → store in AST → resolve during lowering."

### 2.1 Subnet Auto-Assignment on Links

**Problem**: Every /30 link requires manually writing both addresses.

```nll
# Before
link s1:e1 -- l1:e1 { 10.0.11.1/30 -- 10.0.11.2/30 }
# After
link s1:e1 -- l1:e1 { 10.0.11.0/30 }
```

**AST** (`ast.rs`, `LinkDef`): Add `subnet: Option<String>` field.

**Parser** (`parser.rs`, `parse_link()` at line 1020-1028): After parsing the
first CIDR, check if next token is `DashDash`:
- Yes → parse second CIDR (existing behavior, store in `left_addr`/`right_addr`)
- No → store in `subnet` field

**Lowering** (`lower.rs`): When `subnet` is set, compute addresses:

```rust
fn split_subnet(cidr: &str) -> Result<(String, String)> {
    let (network, prefix) = parse_cidr(cidr)?;
    match prefix {
        32 => Err("cannot split /32"),
        31 => Ok((format!("{network}/{prefix}"), format!("{}/{prefix}", next_ip(network)))),
        _  => Ok((format!("{}/{prefix}", next_ip(network)),
                   format!("{}/{prefix}", next_ip(next_ip(network))))),
    }
}
```

**Helpers** (`helpers.rs`): Add `next_ip(addr: IpAddr) -> IpAddr` — increment
by 1 for both IPv4 and IPv6.

### 2.2 List Iteration in For-Loops

**Problem**: For-loops only support integer ranges.

```nll
# Before (awkward)
for i in 1..3 { node server${i} }
# After (readable)
for role in [web, api, db] {
    node ${role} { route default via 10.0.0.1 }
}
```

**AST** (`ast.rs`):

```rust
pub enum ForRange {
    IntRange { start: i64, end: i64 },
    List(Vec<String>),
}
pub struct ForLoop {
    pub var: String,
    pub range: ForRange,  // replaces start/end fields
    pub body: Vec<Statement>,
}
```

**Parser** (`parser.rs`, `parse_for()` at line 1289): After `in`, check for `[`:
- `[` → parse comma-separated values until `]`, return `ForRange::List`
- integer → parse `start..end` (existing), return `ForRange::IntRange`

**Lowering** (`lower.rs`, `expand_for()`):

```rust
let values: Vec<String> = match &for_loop.range {
    ForRange::IntRange { start, end } => (*start..=*end).map(|i| i.to_string()).collect(),
    ForRange::List(items) => items.clone(),
};
for (idx, value) in values.iter().enumerate() {
    vars.insert(for_loop.var.clone(), value.clone());
    vars.insert("loop.index".into(), idx.to_string());
    vars.insert("loop.first".into(), (idx == 0).to_string());
    vars.insert("loop.last".into(), (idx == values.len() - 1).to_string());
    // ... expand body ...
}
```

### 2.3 Default Blocks

**Problem**: MTU and impairment properties repeated on every link.

```nll
defaults link { mtu 9000 }
defaults impair { delay 5ms }

link s1:e1 -- l1:e1 { 10.0.1.0/30 }         # gets mtu 9000
link s1:e2 -- l2:e1 { 10.0.2.0/30 mtu 1500 } # overrides to 1500
```

**Lexer**: Add `Defaults` token.

**AST**: Add `DefaultsDef { kind, link_mtu, impair_props, rate_props }` and
`Statement::Defaults(DefaultsDef)`.

**Lowering**: Track active defaults in `LowerCtx`. When lowering links, merge
defaults with per-link values (per-link wins).

### 2.4 For-Expressions in Lists

**Problem**: Can't generate list values with loops.

```nll
# Before
network mgmt { members [r1:mgmt0, r2:mgmt0, r3:mgmt0, r4:mgmt0] }
# After
network mgmt { members [for i in 1..4 : r${i}:mgmt0] }
```

**Parser**: In list-parsing functions (`parse_members()`, `parse_peers()`, etc.),
check if first token after `[` is `For`. If so, parse
`for var in range : template` and expand immediately.

```rust
fn parse_list_expr(tokens: &[Spanned], pos: &mut usize) -> Result<Vec<String>> {
    expect(tokens, pos, &Token::LBracket)?;
    if check(tokens, *pos, &Token::For) {
        eat(tokens, pos, &Token::For)?;
        let var = expect_ident(tokens, pos)?;
        expect(tokens, pos, &Token::In)?;
        let (start, end) = parse_int_range(tokens, pos)?;
        expect(tokens, pos, &Token::Colon)?;
        let template = parse_name(tokens, pos)?;
        expect(tokens, pos, &Token::RBracket)?;
        Ok((start..=end).map(|i| {
            template.replace(&format!("${{{var}}}"), &i.to_string())
        }).collect())
    } else {
        parse_comma_list(tokens, pos) // existing behavior
    }
}
```

Apply to: `members`, `peers`, `interfaces`, `vlans`, `env`, `volumes`.

### 3.2 Render Command

Add `nlink-lab render <topology.nll>` that parses, expands all loops/variables/
imports, and prints the flat resolved topology.

**New module**: `crates/nlink-lab/src/render.rs`

```rust
pub fn render(topology: &Topology) -> String {
    let mut out = String::new();
    render_lab(&mut out, &topology.lab);
    for (name, profile) in &topology.profiles { render_profile(&mut out, name, profile); }
    for (name, node) in &topology.nodes { render_node(&mut out, name, node); }
    for link in &topology.links { render_link(&mut out, link); }
    for (name, net) in &topology.networks { render_network(&mut out, name, net); }
    // impairments, rate limits...
    out
}
```

**CLI**: Add `Render { topology: PathBuf }` subcommand to clap.

Output is valid NLL that can be re-parsed (round-trip property).
